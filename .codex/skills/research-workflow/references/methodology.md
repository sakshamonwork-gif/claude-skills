# Research Methodology Reference

## Philosophical Foundation

This workflow is grounded in **systematic review methodology** — borrowed from
academic meta-analyses — and adapted for AI-assisted rapid research.

Core principles:
1. **Reproducibility** — The same queries on the same day should yield similar conclusions.
2. **Transparency** — Every claim traces back to a documented source.
3. **Proportionality** — Confidence in a finding is proportional to the breadth and quality of evidence.
4. **Falsifiability** — Actively seek contradicting evidence, not just confirming evidence.

---

## Query Construction Best Practices

### Boolean Operators
- Use `"exact phrase"` for precise terms
- Use `site:example.com` to restrict to authoritative domains
- Use `-keyword` to exclude noise

### Query Progression
```
Broad → Specific → Critical → Verification
```

Never start with hyper-specific queries — broad queries reveal the landscape
and often surface angles you hadn't considered.

### Temporal Framing
Always include a recency query (last 12 months) for fast-moving fields:
```
"[topic] 2025" OR "[topic] 2026"
```

## Cognitive Biases to Guard Against

| Bias | Risk | Mitigation |
|------|------|------------|
| Confirmation bias | Only searching for evidence that supports initial hypothesis | Actively run a "criticism / limitations" query |
| Availability bias | Over-weighting the first result found | Evaluate at least 5–6 sources before concluding |
| Authority bias | Trusting a source because it's well-known | Apply source checklist even to prestigious outlets |
| Recency bias | Ignoring older foundational research | Include at least one "fundamentals" or "history" query |

## Confidence Calibration

```
Single source                  → 🔴 Low
Two independent sources agree  → 🟡 Medium
3+ authoritative sources agree → 🟢 High
Peer-reviewed meta-analysis    → 🟢 High (automatically)
```

Always state confidence level inline when writing findings.
